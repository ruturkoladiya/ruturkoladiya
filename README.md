## 〈 Featured Projects

<details>
<summary><b>⬡ BakeFlow — Smart Order & Inquiry Manager for Bakers</b></summary>

<br/>

A full-stack SaaS-style platform built for home bakers who need a real business tool — not a spreadsheet. BakeFlow gives every baker their own shareable digital storefront, a complete product management system, and a structured order workflow — all without requiring any technical knowledge from the user.

| Attribute | Details |
|-----------|---------|
| **Stack** | React · Node.js · Express · PostgreSQL · TailwindCSS |
| **Scope** | End-to-end: admin panel, customer-facing store, order lifecycle |
| **Architecture** | Decoupled frontend/backend · RESTful API · Role-based flows |
| **UX Focus** | Non-technical user design · Mobile responsive · Zero onboarding friction |
| **Key Features** | Unique shareable store links · Order accept/reject workflow · Product CRUD · Search & filtering |
| **Frontend Repo** | [bake-flow-frontend](https://github.com/ruturkoladiya/bake-flow-frontend) |
| **Backend Repo** | [bake-flow-backend](https://github.com/ruturkoladiya/bake-flow-backend) |
| **Live Demo** | [bake-flow-frontend.vercel.app](https://bake-flow-frontend.vercel.app/) |

**Why this project matters:** Most tools built for small food businesses are either too simple (WhatsApp) or too complex (Shopify). BakeFlow sits in the gap — purpose-built, lightweight, and immediately useful. The real engineering challenge was designing a multi-tenant-style store system with isolated baker contexts under a single backend, while keeping the admin UX approachable for first-time digital users.

</details>

<br/>

<details>
<summary><b>⬡ DreamSpace — Luxury Real Estate Landing Page</b></summary>

<br/>

A high-polish, conversion-focused landing page for a luxury real estate brand. Built to communicate premium positioning through motion, typography, and restrained UI — connecting prospective clients with agents without unnecessary friction.

| Attribute | Details |
|-----------|---------|
| **Stack** | React · TailwindCSS · Framer Motion |
| **Design** | Luxury aesthetic · Fully responsive · Smooth scroll animations |
| **Focus** | Brand credibility · CTA clarity · Mobile-first layout |
| **Performance** | Optimised asset loading · Minimal bundle footprint |
| **Repository** | [luxury-real-estate-landing](https://github.com/ruturkoladiya/luxury-real-estate-landing) |
| **Live Demo** | [luxury-realestate-khaki.vercel.app](https://luxury-realestate-khaki.vercel.app/) |

**Engineering note:** The challenge here was balancing visual richness with performance. Animation libraries can bloat render time quickly on landing pages. The solution was a targeted motion strategy — entrance animations on hero elements only, CSS transitions for repeated interactions — keeping the page feeling premium without sacrificing load speed.

</details>

<br/>

<details>
<summary><b>⬡ BrewStory — Animated Brewery Brand Experience</b></summary>

<br/>

A storytelling-driven brewery landing page where vintage brand identity meets interactive motion. Built to demonstrate how GSAP can be used to craft genuine narrative experiences on the web — not just decoration, but motion as a design language.

| Attribute | Details |
|-----------|---------|
| **Stack** | React · TailwindCSS · GSAP |
| **Design** | Vintage editorial meets modern motion · Scroll-triggered sequences |
| **Focus** | Immersive storytelling · Brand atmosphere · Interaction design |
| **Highlight** | GSAP timeline choreography · Scroll-synced animation |
| **Repository** | [brewstory-landing-gsap](https://github.com/ruturkoladiya/brewstory-landing-gsap) |

**Engineering note:** GSAP timelines were coordinated with scroll position using ScrollTrigger, creating the effect of a narrative unfolding as the user moves down the page. The constraint was keeping animation purposeful — every motion had to serve the story, not just exist.

</details>

<br/>

<details>
<summary><b>⬡ RTK Query CRUD — Open Source Technical Reference</b></summary>

<br/>

A clean, documented implementation of Redux Toolkit Query for full CRUD operations — built as a practical reference for developers learning modern async state management patterns in React.

| Attribute | Details |
|-----------|---------|
| **Stack** | React · Redux Toolkit · RTK Query |
| **Purpose** | Open-source learning resource · Technical reference |
| **Covers** | GET · POST · PUT · DELETE · Cache invalidation · Loading states |
| **Audience** | Frontend developers transitioning to RTK from plain Redux or Context |
| **Repository** | [rtk-query](https://github.com/ruturkoladiya/rtk-query) |

**Note:** RTK Query dramatically reduces boilerplate for data-fetching logic. This repo documents the patterns I use in production projects — cache tags, optimistic updates, and error boundary handling — in a format that's easy to reference and fork.

</details>
